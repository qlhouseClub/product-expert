# 安全策略

## 支持范围

当前 `main` 分支与最新 GitHub Release 受到支持。旧版本中的问题可能要求先升级到最新版本复现。

## 私密报告

未公开修复的问题请使用仓库 **Security → Report a vulnerability** 提交私密报告，不要先创建公开 Issue。报告应尽量包括：

- 受影响的版本、平台与安装方式
- 触发问题的输入、上下文和可用工具
- 实际行为、预期行为和影响范围
- 最小复现步骤、日志或经过脱敏的证据
- 已知缓解方式

请勿在报告、Issue、日志或示例中提交真实 Token、密钥、账号、客户数据或个人信息。

## 安全问题范围

- 技能或适配包泄露凭据、私有数据或未授权内容
- 提示注入导致未经授权的文件、网络、数据库或外部写操作
- 构建产物与仓库源文件不一致或被篡改
- 构建脚本越出 `dist/` 范围删除、覆盖或收集文件
- 依赖、发布流程或安装说明造成供应链风险

单纯的产品观点分歧、输出质量偏好或缺少某项能力通常不属于安全漏洞，可使用普通 Issue 反馈。

## 处理原则

维护者会先确认影响和复现条件，再决定修复、缓解、公告和版本发布方式。问题修复前，请避免公开可直接利用的细节。
